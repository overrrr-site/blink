# UI Design Guidelines

論理的なUIデザイン原則に基づき、直感的でアクセシブル、視覚的に優れたインターフェースを作成する。

## Core Principles

### 1. スペーシング：関連性に基づく配置

要素間の距離で関連性を表現する。

```
spacing-scale:
  xs: 4px   # 密接に関連（アイコンとラベル）
  sm: 8px   # 強く関連（フォーム要素内）
  md: 16px  # 関連あり（セクション内要素）
  lg: 24px  # 弱い関連（セクション間）
  xl: 32px  # 独立（主要セクション間）
  2xl: 48px # 完全分離（ページセクション）
```

**実装ルール：**
- 内側から外側へ、段階的にスペースを増やす
- 8pxグリッドに揃える
- コンテナより余白で関連性を示す（視覚的にシンプル）

### 2. コントラスト比（WCAG 2.1 AA準拠）

| 要素タイプ | 最小コントラスト比 |
|-----------|------------------|
| 通常テキスト（18px以下） | 4.5:1 |
| 大きいテキスト（18px太字/24px以上） | 3:1 |
| UI要素（ボタン、アイコン、境界線） | 3:1 |

**実装：**
```css
/* 推奨カラー例 */
--text-primary: #1a1a1a;    /* 本文：純黒を避ける */
--text-secondary: #525252;   /* 補助テキスト */
--text-tertiary: #737373;    /* 最低限のコントラスト確保 */
--border-default: #d4d4d4;   /* UI境界線 */
```

### 3. ボタン設計

**3段階の視覚的重み：**
```
Primary:   塗りつぶし + 高コントラスト（最重要アクション、画面に1つ）
Secondary: 境界線 + 透明背景（代替アクション）
Tertiary:  テキストのみ（補助的アクション）
```

**サイズ要件：**
- 最小タップターゲット: 48×48px
- ボタン間の最小間隔: 8px
- 頻繁に使用するボタンはより大きく

### 4. タイポグラフィ

**フォント選択：**
- Sans-serifを基本とする（Inter, Noto Sans JP等）
- x-heightが高いフォントを選ぶ（小サイズでの可読性）
- font-weight: 400（regular）と 600-700（bold/semi-bold）のみ使用

**テキストスタイル：**
```css
/* 大見出しはレタースペースを詰める */
h1 { letter-spacing: -0.02em; }

/* 本文は1.5以上の行間 */
body { line-height: 1.5; }

/* 左揃えを基本とする（英語・日本語とも） */
text-align: left;

/* 大文字のみの使用を避ける */
text-transform: none;
```

### 5. 色の使い方

**原則：**
- 色だけで意味を伝えない（色覚多様性への配慮）
- ブランドカラーはインタラクティブ要素に限定
- 装飾目的の色使用を最小限に

**状態表示の実装：**
```css
/* 悪い例：色のみ */
.selected { color: blue; }

/* 良い例：色 + 視覚的手がかり */
.selected { 
  color: blue; 
  border-bottom: 2px solid currentColor; /* 下線追加 */
}

/* アイコンの選択状態 */
.icon-selected { fill: currentColor; } /* 塗りつぶし */
.icon-unselected { fill: none; stroke: currentColor; } /* アウトライン */
```

### 6. 視覚的階層

**スクイントテスト：** デザインをぼかして見ても、最も重要な要素が目立つか確認する。

**階層の作り方：**
1. サイズ：重要な要素を大きく
2. 色/コントラスト：重要な要素を高コントラストに
3. 余白：重要な要素の周りに余白を
4. 位置：重要な要素を目立つ位置に

### 7. 一貫性

**統一すべき要素：**
- アイコンスタイル（線の太さ、塗り/線、角丸）
- 角丸の半径（border-radius）
- シャドウの深さと色
- インタラクティブ要素の見た目

```css
/* デザイントークン例 */
--radius-sm: 4px;
--radius-md: 8px;
--radius-lg: 12px;
/* 全要素で統一して使用 */
```

### 8. ミニマリズム ≠ シンプル

**削除してはいけないもの：**
- アイコンのテキストラベル（意味の明確化）
- 重要なアクション（隠さない）
- アクセシビリティに必要な視覚的手がかり

**削除を検討すべきもの：**
- 装飾的な境界線・背景
- 冗長なコンテナ
- 意味のない色使い

### 9. アイコンとテキストのバランス

アイコンとテキストを並べる際、視覚的な重みを揃える：

```css
.icon-text-pair {
  display: flex;
  align-items: center;
  gap: 8px;
}
.icon { 
  width: 20px; 
  height: 20px;
  color: #525252; /* アイコンと同程度の重みに */
}
.label {
  font-weight: 500; /* テキストを少し太く */
  color: #1a1a1a;
}
```

## Quick Checklist

作成・レビュー時に確認：

- [ ] スペーシングは8pxグリッドに揃っているか
- [ ] 関連要素は近く、無関係な要素は離れているか
- [ ] テキストコントラスト比 ≥ 4.5:1（小テキスト）
- [ ] UIコントラスト比 ≥ 3:1（ボタン、アイコン）
- [ ] プライマリボタンは画面に1つか
- [ ] タップターゲット ≥ 48×48px
- [ ] 色以外の視覚的手がかりがあるか
- [ ] フォントウェイトは2種類以内か
- [ ] 行間 ≥ 1.5（本文）
- [ ] アイコンスタイルは統一されているか
- [ ] 角丸は統一されているか
- [ ] 純黒(#000)を避けているか

## レビュー時の指摘例

```
問題: ボタンのコントラスト比が2.5:1で不足
修正: 背景色を #3b82f6 → #2563eb に変更（コントラスト比 4.8:1）

問題: 選択状態が色のみで表現されている
修正: 下線またはアイコン塗りつぶしを追加

問題: カード間の余白が不均一（12px, 16px, 20px混在）
修正: 全て16px（md）に統一
```
