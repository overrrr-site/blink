# 動画素材用スクリーンショット撮影ガイド

## 事前準備

### 1. デモデータ投入
```bash
cd server && npx tsx src/db/seed-demo.ts
```

### 2. ローカルサーバー起動
```bash
npm run dev
```

### 3. ブラウザ設定
- Chrome DevTools → デバイスツールバー → **iPhone 14 Pro (390x844)** でモバイル撮影
- デスクトップは **1280x800** 推奨（MacBook Air サイズ）
- 管理画面: `http://localhost:5173`
- LIFF: `http://localhost:5173/liff`

---

## 撮影シート

### ① ダッシュボード（管理画面 1画面目）
**URL:** `http://localhost:5173/`
**見せたいもの:**
- 今日の予約 6件（登園済3件 + 予定3件）のカード
- 時間帯ごとのグルーピング（9:00, 9:30, 10:00, 11:00, 13:00）
- ワクチン期限アラート（ベルアイコン → モーダル）
- 未入力日誌の件数バッジ
- 点検記録・お知らせのクイックアクションカード

**撮影ポイント:**
- アラートベルをクリックしてモーダルを開いた状態も撮影
- ステータスフィルタ「すべて」が選択された状態で全体が見える構図

---

### ② 犬一覧（管理画面 2画面目）
**URL:** `http://localhost:5173/customers` → 「ワンちゃん」タブ切替
**見せたいもの:**
- 10匹の犬が並んでいるリスト
- 写真 + 犬名 + 犬種 + 飼い主名
- 検索バー（空の状態で全件表示）
- 統計カード（飼い主 X人 / ワンちゃん X匹）

**撮影ポイント:**
- 「ワンちゃん」タブが選択された状態
- スクロールせずに 5-6匹が見える状態

---

### ③ 犬詳細（管理画面 3画面目）
**URL:** `http://localhost:5173/dogs/{id}` （コロ or レオの ID）
**見せたいもの:**
- ヘッダー: 犬写真 + 名前 + 犬種 + 年齢
- 基本情報セクション
- 健康情報（ワクチン日付 + ステータスインジケーター）
- 性格・特性セクション
- 契約・残回数サマリー

**撮影ポイント:**
- 情報が充実している犬を選ぶ（コロ: ゴールデンレトリーバー推奨）
- 全セクションが「ある程度見える」構図（全部読めなくてOK）
- ワクチンアラート（黄色/赤）が表示されている状態

---

### ④ 予約カレンダー（管理画面 4画面目）
**URL:** `http://localhost:5173/reservations`
**見せたいもの:**
- 月表示カレンダー
- 予約が入っている日にバッジ表示（件数）
- 今日がハイライトされている
- 日付をクリックした時のサイドパネル（予約詳細リスト）

**撮影ポイント:**
- 今日の日付をクリックして、サイドパネルに予約リストが表示された状態
- カレンダーに適度にバッジが散らばっている月を選ぶ

---

### ⑤ 日誌作成 - チェック前（管理画面 5画面目）
**URL:** `http://localhost:5173/journals/create/{予約ID}`
（ダッシュボードの登園済カードから「日誌」ボタン → 遷移）
**見せたいもの:**
- Step 1（写真）は飛ばして Step 3（詳細）を表示
- トレーニング項目チェックリスト（おすわり、まて、おいで...）
- トイレ記録（午前/午後）
- 未チェック状態（まだ何も選択していない）

**撮影ポイント:**
- 「3. 詳細」ステップが選択された状態
- チェック項目が並んでいる画面全体

---

### ⑥ 日誌作成 - AI生成後（管理画面 6画面目）
**URL:** 同上（⑤の続き）
**見せたいもの:**
- Step 2（コメント）を表示
- AI生成されたコメントがテキストエリアに表示されている
- 写真がアップロードされた状態

**撮影方法:**
- ⑤の画面でチェック項目をいくつか選択
- Step 2 に戻り、AI生成ボタンを押す（要ネットワーク接続）
- または、手動でコメント欄にデモテキストを貼り付けて撮影

**デモコメント例:**
```
今日は他のお友達とたくさん遊びました！特にボール遊びが楽しかったようで、
しっぽを振りながら走り回っていました。「おすわり」と「まて」はしっかりできて、
褒められると嬉しそうにしていました。トイレも午前・午後ともに成功です。
午後はぐっすりお昼寝をして、夕方にはまた元気に遊んでいました。
```

---

### ⑦ LIFF ホーム（飼い主 7画面目）
**URL:** `http://localhost:5173/liff/` → ログイン
**見せたいもの:**
- 次回予約カード（今日の予約が表示）
- 「登園する」ボタン
- メニューカード（予約、日誌、マイページ、お知らせ）

**撮影ポイント:**
- DevTools でモバイルサイズに設定（iPhone 14 Pro）
- ログインは開発モード（dev-user）で自動ログイン
- 予約カードに「本日」バッジが表示されている状態

---

### ⑧ LIFF 日誌詳細（飼い主 8画面目）
**URL:** `http://localhost:5173/liff/journals/{id}`
（LIFF ホーム → 日誌メニュー → 日誌一覧 → 詳細）
**見せたいもの:**
- 今日の様子コメント（AI生成風の自然な文章）
- 写真（2枚）
- トレーニング結果（○△ 表示）

**撮影ポイント:**
- モバイルサイズで撮影
- コメントが全文見える構図
- 写真が2枚表示されている

---

### ⑨ 日誌履歴一覧（管理画面 9画面目）
**URL:** `http://localhost:5173/journals`
**見せたいもの:**
- 日付ごとにグルーピングされた日誌一覧
- 犬写真 + 犬名 + 担当スタッフ + コメント抜粋
- 犬フィルタチップ（横スクロール）
- 検索バー

**撮影ポイント:**
- 日付グループが 2-3 個見える構図
- 犬フィルタが横に並んでいる状態

---

## デモデータの削除

撮影が終わったらデモデータを削除:
```bash
cd server && npx tsx src/db/seed-demo.ts --rollback
```

## 補足: 効率的な撮影フロー

1. デスクトップサイズ（1280x800）で管理画面 ①②③④⑤⑥⑨ を連続撮影
2. DevTools でモバイルサイズに切替えて ⑦⑧ を撮影
3. 各画面で Mac のスクリーンショット (`Cmd+Shift+4`) を使用
4. 動画用素材なら QuickTime で画面収録（`Cmd+Shift+5`）
