# UI/UXデザインレビュー対象ページ一覧

このドキュメントは、UI/UXデザインレビューを行うべきページ（コンポーネント）を優先度別にリストアップしたものです。

## 優先度: 高（最重要レビュー対象）

### スタッフ向けWebアプリ

#### 1. **ダッシュボード（メイン画面）**
- **ファイル**: `client/src/pages/Dashboard.tsx`
- **レビュー観点**:
  - フィルター機能（すべて/登園前/利用中/帰宅済）の使いやすさ
  - 予約カードの情報表示と展開/折りたたみUI
  - チェックイン操作の分かりやすさ
  - 未完了日誌の表示とアクション
  - アラート表示の見やすさ
  - 空状態（予約がない場合）の表示

#### 2. **日誌作成画面**
- **ファイル**: `client/src/pages/JournalCreate.tsx`
- **レビュー観点**:
  - ステップバイステップ（写真→コメント→詳細）の進行感
  - 写真アップロード・プレビュー・削除の操作性
  - AIコメント生成ボタンの配置とフィードバック
  - メモ書き入力エリアの使いやすさ
  - トレーニング記録の入力方法（○/△/−選択）
  - トイレ記録の入力UI
  - 写真解析結果の表示と活用方法

#### 3. **予約作成・編集画面**
- **ファイル**: `client/src/pages/ReservationCreate.tsx`
- **レビュー観点**:
  - ステップインジケーターの分かりやすさ
  - 日付・時間選択のUI
  - 犬選択の検索・フィルター機能
  - フォーム入力のバリデーションとエラー表示

#### 4. **顧客一覧・検索画面**
- **ファイル**: `client/src/pages/Customers.tsx`
- **レビュー観点**:

#### 5. **設定画面**

## レビュー時のチェックポイント

### 全体的な観点

1. **アクセシビリティ**
   - キーボード操作の対応
   - スクリーンリーダー対応
   - コントラスト比
   - タップターゲットサイズ（最小44x44px）

2. **レスポンシブデザイン**
   - モバイル（375px〜）での表示
   - タブレット（768px〜）での表示
   - デスクトップ（1024px〜）での表示

3. **パフォーマンス**
   - ローディング状態の表示
   - エラー状態の表示
   - 空状態の表示

4. **一貫性**
   - デザインシステムの統一
   - アイコンの使い方
   - カラーパレットの統一
   - タイポグラフィの統一

5. **ユーザビリティ**
   - 操作の直感性
   - エラーメッセージの分かりやすさ
   - 成功時のフィードバック
   - 確認ダイアログの必要性

## レビュー実施方法

1. **優先度: 高**のページから順にレビューを実施
2. 各ページについて、上記のチェックポイントを確認
3. 問題点をリストアップし、改善提案を作成
4. デザインシステムの統一性を確認
5. モバイル・タブレット・デスクトップでの表示を確認
